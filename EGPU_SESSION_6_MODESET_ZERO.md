# Sesión 6 - Fix arranque con modeset=0 (2 Mayo 2026, 01:32-01:34)

## Problema

El arranque normal (sin recovery) se quedaba en pantalla negra justo antes del escritorio. Causa: `nvidia-modeset` se cargaba automáticamente con `modeset=1` por el archivo `/etc/modprobe.d/nvidia-graphics-drivers-kms.conf` generado por el paquete nvidia-driver.

## Fix aplicado

```bash
sudo sed -i 's|modeset=1|modeset=0|g' /etc/modprobe.d/nvidia-graphics-drivers-kms.conf
sudo update-initramfs -u -k 6.11.0-19-generic
```

Contenido final de `/etc/modprobe.d/nvidia-graphics-drivers-kms.conf`:
```
# This file was generated by nvidia-driver-535
# Set value to 0 to disable modesetting
options nvidia-drm modeset=0
```

## Nota importante

Este archivo es generado automáticamente por el paquete `nvidia-driver-*`. Si se actualiza el driver, puede sobreescribirse y volver a `modeset=1`, causando el cuelgue de nuevo. Hay que vigilarlo tras actualizaciones.
